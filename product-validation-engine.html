Decision Logic — Exact Rules Applied
KILL PRODUCT — any of these
• CTR < threshold (bad ad or product)
• Profit per unit < R300 after all costs
• CPP > break-even CPA
• Conv. rate < minimum AND CPP > max
• Health score < 35
TEST MORE — borderline signals
• CTR ≥ threshold but conv. rate low
• High ATC but low purchases (checkout issue)
• Profitable but not enough data (<10 purchases)
• Health score 35–69
• CPP within 20% of break-even
SCALE PRODUCT — all must be true
• CTR ≥ threshold
• Conv. rate ≥ threshold
• CPP < max CPP threshold
• Profit per unit ≥ R300
• Health score ≥ 70
• 10+ purchases (enough data)

System Architecture
Data Input Layer
This form + localStorage now. Future: Meta Ads API, TikTok API, Shopify Webhooks → auto-populate fields every 6 hours.
Metrics Engine
Pure JS functions. All formulas run client-side. Backend version: Python/FastAPI endpoint /api/metrics/{product_id}.
Decision Engine
Rule engine runs after every data update. Can be moved to server for scheduled auto-evaluation and Slack/email alerts.
Storage
localStorage for offline use. Upgrade path: PostgreSQL + Supabase (free tier) for cloud sync across devices.
Metrics Formulas Reference
// All formulas used in this system CTR = clicks / impressions × 100 CPC = spend / clicks CVR = purchases / clicks × 100 ATC_rate = atc / clicks × 100 checkout_rate = ic / atc × 100 CPP = spend / purchases rev_per_purch = revenue / purchases gateway_cost = sell_price × (gateway_pct / 100) total_unit_cost = cogs + shipping + customs + packaging + other + gateway_cost profit_per_unit = sell_price - total_unit_cost total_profit = (profit_per_unit × purchases) - spend // Break-even CPA breakeven_cpa = sell_price - total_unit_cost // Max you can spend per purchase and still profit // Health score (0–100) score = 0 if CTR >= t_ctr: score += 25 if CVR >= t_cvr: score += 25 if CPP <= t_cpp: score += 25 if profit >= t_profit: score += 25
